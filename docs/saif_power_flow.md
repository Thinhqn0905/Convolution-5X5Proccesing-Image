# SAIF Power Flow (D455 Stream -> Vivado Power)

## Why this flow
- `read_vcd` parsing is fragile across simulators and can fail unexpectedly.
- XSIM-generated SAIF has better compatibility with Vivado `report_power`.
- Activity comes from real D455 frames, not synthetic toggles.

## Recommended commands
Run from repository root:

```powershell
.\scripts\run_power_with_saif.ps1 `
  -FrameHex .\captures\d455\full640_smoke\hex_in\frame_000000.hex `
  -Kernel gaussian5 `
  -Width 640 `
  -Height 480 `
  -SaifOut .\sim\activity.saif `
  -VivadoBat E:\Vivado\2023.2\bin\vivado.bat
```

Clock/frequency sweep with SAIF:

```powershell
.\scripts\sweep_clock_with_saif.ps1 `
  -PeriodsNs @(30.0, 27.0, 25.0) `
  -FrameHex .\captures\d455\full640_smoke\hex_in\frame_000000.hex `
  -Kernel gaussian5 `
  -Width 640 `
  -Height 480 `
  -SaifOut .\sim\activity.saif `
  -VivadoBat E:\Vivado\2023.2\bin\vivado.bat
```

## Fail-fast behavior (path error handling)
- Both scripts now run in strict mode for Vivado output.
- If output contains `The system cannot find the path specified.`:
  - Script aborts the current run immediately.
  - Script attempts one automatic repair pass by regenerating the project.
  - If still present, script exits with explicit error and log location.

Logs generated by strict mode:
- `vivado_project/reports/vivado_power_with_saif.log`
- `vivado_project/reports/vivado_sweep_<period>ns.log`
- `vivado_project/reports/vivado_path_repair.log` (only when repair was attempted)

## Validate SAIF application in report
Check these lines in power report/log:
- `Simulation Activity File | ...activity.saif`
- `Design Nets Matched      | ...`
- `Overall confidence level | ...`
